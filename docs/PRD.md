## 1. 제품 개요

### 1.1 한 줄 요약

**함께 여행을 계획하고, 실시간으로 협업하며, AI가 도와주는 여행 플래너 웹앱**

### 1.2 문제 정의

| 문제 | 현재 사용자 행동 | 고통 포인트 |
|------|----------------|------------|
| 여행 정보가 흩어짐 | 카톡, 메모, 블로그 북마크 등에 분산 저장 | 정보 유실, 중복 검색, 공유 어려움 |
| 협업이 번거로움 | 카톡방에서 링크 공유 → 의견 수렴 | 결정이 늦어지고, 대화에 묻힘 |
| 일정 조율이 어려움 | 엑셀/구글시트로 일정 수동 관리 | 동선 비효율, 수정 충돌 |
| 예산 관리가 안 됨 | 여행 후 정산을 따로 계산 | 실시간 지출 파악 불가 |
| 정보 입력이 귀찮음 | 숙소/맛집 URL을 하나씩 복붙하고 정리 | 반복 작업, 시간 낭비 |

### 1.3 솔루션

하나의 앱에서 **장소 수집 → 투표 → 일정 편성 → 예산 관리 → 여행 후기**까지 전 과정을 커버하는 협업형 여행 플래너. AI가 추천과 일정 생성을 보조하고, URL 공유만으로 장소 정보가 자동 수집된다.

### 1.4 타겟 사용자

| 페르소나 | 특징 | 핵심 니즈 |
|----------|------|----------|
| **기획자 친구** | 여행 일정을 주도적으로 짜는 사람 | 효율적인 일정 편성, 동선 최적화 |
| **참여자 친구** | 링크 공유하고 투표에 참여 | 쉬운 장소 공유, 간편한 투표 |
| **커플/소그룹** | 2~6명 규모의 여행 | 실시간 의사결정, 예산 분담 |
| **솔로 여행자** | 혼자 여행 계획 | AI 추천, 체계적 정리 |

## 2. 핵심 기능 (Features)

### 2.1 기능 맵

```
여행 플래너
├── 🏠 여행 관리
│   ├── 여행 생성/수정/삭제
│   ├── 멤버 초대 (초대 코드)
│   └── 역할 관리 (admin/editor/viewer)
│
├── 📍 장소 관리
│   ├── 수동 등록 (폼 입력)
│   ├── Google Places 검색
│   ├── URL 공유로 자동 등록 (PWA Share Target)
│   ├── 장소 투표 (좋아요/비교)
│   └── 지도 보기
│
├── 📅 일정 관리
│   ├── 캘린더 뷰
│   ├── 타임라인 뷰
│   ├── 드래그앤드롭 정렬
│   └── 장소-일정 연결
│
├── 💰 예산 관리
│   ├── 통화별 예산 설정
│   ├── 지출 기록
│   └── 멤버 간 정산
│
├── 📝 여행 후기
│   ├── 날짜별 후기 작성
│   └── 사진 업로드
│
├── 🤖 AI 어시스턴트
│   ├── 장소/활동 추천
│   ├── 일정 자동 생성
│   ├── 동선 검토
│   └── 빈 시간대 채우기
│
└── ⚡ 실시간 협업
    ├── 온라인 멤버 표시
    ├── 활동 알림 (토스트)
    └── 실시간 데이터 동기화
```

## 3. 사용자 플로우 (User Flow)

### 3.1 핵심 플로우: 여행 생성 → 완료

```
[회원가입/로그인]
       ↓
[대시보드: 내 여행 목록]
       ↓
[+ 여행 만들기] → 제목, 목적지, 날짜 입력
       ↓
[여행 상세 페이지]
       ↓
  ┌────────┬────────┬────────┬────────┬────────┐
  ↓        ↓        ↓        ↓        ↓        ↓
[장소]   [일정]   [예산]   [후기]   [멤버]   [AI]

[장소 등록]                               [AI 추천]
 ├ 검색                                    ├ 맛집
 ├ URL 공유                                ├ 활동
 └ 수동 입력                               └ 일정 생성
       ↓
[장소 투표] → 좋아요/비교
       ↓
[일정 편성] → 캘린더/타임라인 + DnD
       ↓
[예산 설정] → 지출 기록 → 정산
       ↓
[여행 출발!] → 실시간 일정 확인
       ↓
[여행 후기] → 날짜별 사진/글
```

### 3.2 핵심 플로우: 멤버 초대

```
[여행 생성자] → [멤버 탭] → [초대 링크 복사]
                                    ↓
                        [카톡/문자로 링크 공유]
                                    ↓
[초대받은 사람] → /join/[inviteCode] → 로그인 → 자동 참가
```

### 3.3 핵심 플로우: URL 공유 (모바일)

```
[모바일 브라우저에서 숙소 페이지 발견]
              ↓
[공유 버튼 → "여행 플래너" 선택]
              ↓
[PWA Share Target] → URL + 제목 수신
              ↓
[최근 여행 자동 선택] (30분 내 Sticky Context)
              ↓
[즉시 저장 (enriched=false)]
              ↓
[백그라운드 풍부화]
 ├ URL 파싱 (Booking, Agoda 등 12개 사이트)
 ├ Google Places API 매칭
 └ 메타데이터 자동 채움 (주소, 사진, 평점, 가격)
```

## 4. 비기능 요구사항 (Non-Functional Requirements)

### 4.1 성능

| 항목 | 목표 | 현재 상태 |
|------|------|----------|
| 초기 로딩 (LCP) | < 2.5s | 미측정 |
| 페이지 전환 | < 300ms | App Router로 대응 |
| API 응답 (일반) | < 500ms | Supabase 직접 쿼리 |
| AI 응답 (스트리밍) | 첫 토큰 < 1s | Gemini API |
| 동시 접속자 | 여행당 10명 | Supabase Realtime |

### 4.2 접근성

| 항목 | 목표 | 현재 상태 |
|------|------|----------|
| 반응형 | 모바일(360px) ~ 데스크톱(1440px) | ✅ 구현 |
| PWA 설치 | iOS Safari, Android Chrome | ✅ 매니페스트 |
| 오프라인 | 기본 읽기 가능 | ✅ Service Worker |

### 4.3 보안

| 항목 | 구현 상태 |
|------|----------|
| 인증 | Supabase Auth (OAuth) |
| 권한 | trip_members 기반 역할 체크 |
| SSRF 방어 | DNS 해석 + 사설 IP 차단 |
| Rate Limiting | 사용자당 분당 10 요청 (Share API) |
| 입력 검증 | URL 프로토콜/도메인/길이 검증 |

## 8. 용어 정의 (Glossary)

| 용어 | 정의 |
|------|------|
| **Trip** | 하나의 여행 계획 단위 |
| **Place** | 여행 중 방문할 장소 (숙소/관광지/맛집/기타) |
| **Schedule** | 특정 날짜의 일정 컨테이너 |
| **ScheduleItem** | 일정 내 개별 항목 (시간대, 장소 연결) |
| **Budget** | 여행 전체 예산 (통화별) |
| **Expense** | 개별 지출 기록 |
| **Settlement** | 멤버 간 정산 내역 |
