## 권장 진행 순서

현재 MVP 코드가 80% 완성된 상태이므로, 일반적인 순서(Phase 0→7)가 아니라
**빠진 기초를 채우면서 배포를 향해 가는 순서**를 권장합니다:

### 1순위: 기반 다지기 (1~2주)

```
Phase 0-1: 경쟁 분석 (시장에서 포지셔닝 확인)
Phase 3-1: ESLint/Prettier 설정 (코드 품질 기반)
Phase 3-2: Git 워크플로 정립 (브랜칭, 커밋 컨벤션)
Phase 4-3: 에러 핸들링 통일 (코드 안정성)
```

### 2순위: 디자인 & 누락 기능 (2~3주)

```
Phase 2-4: 와이어프레임 → UI 디자인 (현재 화면 스크린샷 기반)
Phase 2-5: 디자인 시스템 정리 (색상, 타이포, 컴포넌트)
Phase 4-2: 누락 기능 보완 (삭제, 수정, 필터 등)
Phase 1-2: 유저 스토리 보완
```

### 3순위: 테스트 & 배포 준비 (2~3주)

```
Phase 5-1: 테스트 환경 구축
Phase 5-2: 핵심 테스트 작성 (Unit + Integration)
Phase 3-3: CI/CD 파이프라인 구축
Phase 6-2: 보안 점검 (배포 전 필수)
```

### 4순위: 배포 & 출시 (1주)

```
Phase 6-1: Vercel 배포
Phase 6-3: 성능 점검
Phase 6-4: 출시 준비 (OG태그, 이용약관 등)
Phase 7-1: 모니터링 연결
```
